## Completion rules

The user-authorized relaxed profile is complete when
`falsirag release solo-paper-readiness` reports `ready:true`, the dedicated
`scripts/solo_paper_release_check.sh` gate passes, and the paper retains every
required limitation. Strict AAAI completion
still requires every strict row above to be closed by its named evidence. The
2+4 profile is a separate, preregistered single-author evidence tier: it is complete
only if its active G-A branch and every subsequently authorized G-K/G-S/release/
readiness gate close with fingerprinted artifacts. If Round 2 G-A fails, faithful
completion means enforcing the registered stop and downgrade branch, not bypassing
it with jury labels. These profiles must never be conflated. In particular:

The accepted no-human redirection profile additionally closes when the tracked
P6-M verifier is valid and the negative outcome is retained without human-label
claims. Its closure does not change the strict human packet's
`ready_to_analyze=false` state; that packet is an inactive optional protocol,
not a remaining task for the accepted profile.

1. LLM preannotations may reduce reviewer effort but cannot be reported as two
   independent human annotations or Cohen's kappa.
2. The review-priority CSV is a queue for scarce future review time, not a
   corrected label file.
3. Local access to the test labels is not an externally held blind test.
4. Smoke, partial, and train runs cannot populate an empirical table. Complete
   development runs may populate only a table explicitly captioned as a
   machine-seeded, single-model development diagnostic, never a blind test.
5. The typed-control contribution survives only if the preregistered paired
   typed-versus-untyped comparison supports it; otherwise the paper must report
   the diagnostic or negative result.
6. `paper/aaai27/ReproducibilityChecklist.tex` must be re-audited after final
   runs so every `partial` or `no` answer remains truthful.
